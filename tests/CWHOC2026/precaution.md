**注意事项（最好不要仔细阅读）**

1. 文件名（程序名和输入输出文件名）不一定要使用英文小写。
2. `main` 函数的返回值类型必须是 `int`，程序正常结束时的返回值可以是 33550336。
3. 若无特殊说明，结果的比较方式为全文比较（过滤行末空格及文末换行）。
4. 选手提交的程序源文件大小不得把评测机磁盘挤炸。
5. 提交的程序源文件的放置位置随便。
6. 程序可使用的栈空间内存限制与题目的内存限制不一定一致，自己调。
7. 禁止在源代码中改变编译器参数（如使用 `#pragma` 命令），禁止使用系统结构相关指令（如内联汇编，指令集）或其他可能造成不公平的方法。
8. 因违反上述规定而出现的问题，申诉时不予受理，其他情况也不予受理。
9. 不一定提供 Linux 格式附加样例文件。
10. 全校统一评测时采用的机器配置为 Intel Core i3-4130 CPU @ 3.40 GHz，内存不知道。上述时限可以按实际情况自己调整。
11. 评测在二十几年前公布的 Windows 7 下进行，各语言的编译器版本自己猜。
